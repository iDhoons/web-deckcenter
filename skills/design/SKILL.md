# Design Principles (디자인 원칙)

UI/컴포넌트 작업 시 다음 원칙을 적용합니다.

---

## 1. 황금비율 & 스케일 (Golden Ratio)

**원칙**: 1:1.618 비율은 자연스러운 아름다움을 만든다.

**Urizen 스페이싱 스케일** (피보나치 유사 패턴):
```
--spacing-xs:  0.25rem (4px)
--spacing-sm:  0.5rem  (8px)
--spacing-md:  1rem    (16px)  ← 기준
--spacing-lg:  1.5rem  (24px)
--spacing-xl:  2rem    (32px)
--spacing-2xl: 3rem    (48px)
--spacing-3xl: 4rem    (64px)
```

**적용**:
- 요소 간 간격은 스케일 내에서 선택
- 작은 요소 → 작은 간격, 큰 섹션 → 큰 간격
- 비율: 내부 padding < 외부 margin

---

## 2. 시각적 계층구조 (Visual Hierarchy)

**원칙**: 크기, 색상, 위치로 중요도를 표현한다.

**Urizen 타이포 스케일**:
```
h1: 3.5rem (56px) - 800 weight - 페이지 제목
h2: 2.5rem (40px) - 700 weight - 섹션 제목
h3: 1.35rem (22px) - 600 weight - 서브 제목
body: 1rem (16px) - 400 weight - 본문
small: 0.875rem (14px) - 보조 텍스트
```

**적용**:
- 한 화면에 h1은 하나만
- 제목 → 부제목 → 본문 순서로 시선 유도
- 중요한 정보는 왼쪽 상단 (F-패턴)
- CTA 버튼은 시각적으로 가장 눈에 띄게

---

## 3. 여백의 미 (White Space)

**원칙**: 여백은 비어있는 것이 아니라 숨 쉬는 공간이다.

**적용**:
- 섹션 간: `--spacing-2xl` ~ `--spacing-3xl` (48-64px)
- 카드 내부: `--spacing-lg` ~ `--spacing-xl` (24-32px)
- 텍스트 블록: `--spacing-md` (16px)
- 줄 간격: `line-height: 1.7` (본문)

**주의**:
- 요소를 빽빽하게 채우지 않는다
- 여백이 부족하면 답답함, 과하면 산만함
- 그룹 내부 < 그룹 외부 간격

---

## 4. 그리드 & 정렬 (Grid & Alignment)

**원칙**: 보이지 않는 선에 맞춰 정렬한다.

**Urizen 그리드**:
```css
max-width: 1200px (--max-width)
기준 단위: 8px (0.5rem)
카드 그리드: repeat(auto-fit, minmax(280px, 1fr))
```

**적용**:
- 모든 수치는 8의 배수 권장 (8, 16, 24, 32, 48, 64...)
- 좌우 정렬 일관성 유지
- 텍스트는 왼쪽 정렬 기본 (한국어/영어)
- 가운데 정렬은 제목, 히어로, CTA에만

---

## 5. 색상 이론 (Color Theory)

**원칙**: 60-30-10 법칙으로 조화를 만든다.

**Urizen 컬러**:
```
60% 배경: --color-bg (#FCFBF8)
30% 텍스트: --color-text (#333333)
10% 강조: --color-accent (#004250)
```

**적용**:
- 강조색은 CTA, 링크, 중요 정보에만
- 배경색 변화로 섹션 구분 (`--color-bg-alt`)
- 다크모드: 색상 반전이 아닌 톤 조정

**다크모드 팔레트**:
```
배경: #0f0f0f → #1a1a1a
텍스트: #e5e5e5
강조: #00a3cc (밝은 틸)
```

---

## 6. 타이포그래피 (Typography)

**원칙**: 가독성이 최우선이다.

**Urizen 폰트**: Paperlogy (한글/영문)

**적용**:
- line-height: 1.7 (본문), 1.15 (제목)
- letter-spacing: -0.02em (제목), normal (본문)
- 한 줄 최대 65-75자 (가독성 최적)
- 폰트 웨이트는 400, 600, 700, 800만 사용

**피해야 할 것**:
- 얇은 폰트 (300 이하) - 가독성 저하
- 모든 대문자 텍스트 - 한글에 부적합
- 너무 작은 폰트 (14px 미만)

---

## 7. 대비 & 균형 (Contrast & Balance)

**원칙**: 대비는 주목, 균형은 안정을 만든다.

**적용**:
- 큰 이미지 ↔ 작은 텍스트
- 어두운 영역 ↔ 밝은 영역
- 복잡한 요소 ↔ 단순한 요소
- 비대칭 균형: 완벽한 대칭보다 동적

**시각적 무게**:
```
높음: 큰 크기, 어두운 색, 복잡한 형태, 이미지
낮음: 작은 크기, 밝은 색, 단순한 형태, 텍스트
```

---

## 8. 근접성 원칙 (Proximity)

**원칙**: 가까운 것은 관련있고, 먼 것은 별개다.

**적용**:
- 라벨과 입력 필드: `--spacing-xs` (4px)
- 폼 필드 간: `--spacing-md` (16px)
- 카드 내 요소: `--spacing-sm` ~ `--spacing-md`
- 섹션 간: `--spacing-2xl` ~ `--spacing-3xl`

**그룹핑**:
```
[제목 + 설명] ← 가깝게
    ↕ 멀리
[버튼 그룹]
```

---

## 9. 반복 & 일관성 (Repetition)

**원칙**: 패턴의 반복이 통일감을 만든다.

**적용**:
- 같은 유형의 요소는 같은 스타일
- CSS 변수 일관 사용 (하드코딩 금지)
- 컴포넌트 재사용
- 호버 효과 통일: `translateY(-8px)` + 그림자 강화

**Urizen 공통 패턴**:
```css
border-radius: 12px (카드), 4-6px (버튼)
transition: all 0.3s ease
box-shadow: var(--shadow-sm) → var(--shadow-lg) on hover
```

---

## 10. 접근성 (Accessibility)

**원칙**: 모두가 사용할 수 있어야 아름답다.

**적용**:
- 텍스트 대비율: 4.5:1 이상 (WCAG AA)
- 포커스 상태: `outline: 2px solid var(--color-accent)`
- 터치 타겟: 최소 44x44px
- `alt` 텍스트 필수

**Urizen 접근성 색상**:
```
--color-text-light: #595959 (WCAG AA 준수)
다크모드 텍스트: #e5e5e5 (충분한 대비)
```
